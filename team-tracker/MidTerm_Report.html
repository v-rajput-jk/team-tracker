Abstract

The Team Performance Tracker & Management System is an enterprise-grade, full-stack web application designed to provide real-time workforce analytics, task management, attendance monitoring, overtime tracking, and AI-driven performance insights for organisational team management. The system integrates directly with Microsoft SharePoint via the Microsoft Graph API using delegated authentication through MSAL (Microsoft Authentication Library), enabling secure, seamless synchronisation of live organisational data.

The front-end is built using React 19 with TypeScript, leveraging Vite as the build tool for optimal development performance. The application features a modern, responsive UI with dark/light theme support, glassmorphism design elements, and smooth animations powered by Framer Motion. Data visualisation is handled through Recharts, providing interactive charts and analytics dashboards.

The back-end is powered by Node.js with Express.js, serving as a secure proxy layer between the React client and Microsoft Graph API. It handles JWT token validation, CORS management, and all SharePoint CRUD operations including employee data retrieval, task/project management, attendance records, and overtime schedule updates.

Key features include a Performance Orbit visualisation for real-time team momentum, an AI Executive Summary generator, a Leaderboard system with automated scoring algorithms, a comprehensive Schedule Management module, multi-language support, and PDF report generation capabilities. The system is designed to replace manual spreadsheet-based tracking with an automated, intelligent, and visually rich management dashboard.

What Is Already Done

1. System Architecture & Setup

The complete project architecture has been designed and implemented as a full-stack application with clear separation of concerns:

  • Front-End (React + TypeScript + Vite): Initialised with Vite build tool, React 19, TypeScript 6, and TailwindCSS 4 for styling. The React Compiler is enabled for optimised rendering.
  • Back-End (Node.js + Express): A RESTful API server running on port 5000, handling all Microsoft Graph API interactions securely.
  • Authentication (MSAL): Azure AD B2C integration with MSAL Browser and MSAL React for delegated OAuth 2.0 authentication flow.

2. Core Modules Completed

2.1 Dashboard Module

A comprehensive analytics dashboard featuring six KPI cards (Total Members, Active Members, Tasks Today, Completed Tasks, In-Progress Tasks, and Team Efficiency), an interactive area chart for KPI performance trends, a Performance Orbit visualisation, and AI Smart Insights with proactive recommendations.

2.2 Task & Project Management Module

Full CRUD operations for project tracking with fields including project name, allocated staff, status, start/end dates, folder links, and remarks. Supports real-time bi-directional sync with SharePoint's Project_Tracking_all list. Features include inline editing, status badges, priority indicators, and progress bars.

2.3 Attendance Management Module

Supports both standard and pivoted SharePoint data formats. Parses attendance statuses (Present, Absent, Late, Half-Day) and maps employee records via column-name decoding for SharePoint's internal hex-encoded field names.

2.4 Overtime & Schedule Management Module

Real-time overtime tracking with editable schedule grids. Supports pivoted column formats where each date is a separate SharePoint column. Features include inline value editing with instant local state updates and background SharePoint PATCH operations.

2.5 Performance Analytics Module

Automated performance scoring algorithm considering task completion rate, attendance consistency, overtime patterns, and deadline adherence. Individual employee profile pages with detailed breakdowns.

2.6 Leaderboard System

Gamified ranking system with automated scoring, tier classifications (Top Tier, High Performer, etc.), and comparative analytics across all team members.

2.7 Authentication & Security

Complete Microsoft Azure AD integration with MSAL for SSO. JWT token validation on the backend with JWKS verification. CORS policies restricting access to localhost origins during development. Authenticated and Unauthenticated route templates.

What Is Planning

1. Upcoming Feature Development

1.1 Role-Based Access Control (RBAC)

Implementation of granular permission levels (Admin, Manager, Team Lead, Employee) to restrict module access and editing capabilities based on user roles fetched from Azure AD group memberships.

1.2 Real-Time Notifications & Alerts

Integration of push notifications for task assignments, deadline reminders, attendance anomalies, and overtime threshold alerts using Web Socket connections or Server-Sent Events.

1.3 Advanced AI/ML Analytics

Incorporation of machine learning models for predictive analytics including burnout risk prediction, project delivery forecasting, optimal resource allocation recommendations, and automated workload balancing suggestions.

1.4 Leave Management Module

A dedicated leave application and approval workflow integrated with the existing attendance system. Features will include leave balance tracking, approval chains, and calendar integration.

1.5 Client Feedback Integration

A feedback collection module allowing clients to rate project deliverables, which feeds directly into the performance scoring algorithm for a more holistic evaluation.

1.6 Mobile Progressive Web App (PWA)

Converting the application into a PWA with offline capabilities, enabling field employees to log attendance and view schedules without constant internet connectivity.

2. Technical Improvements Planned

  • Database Layer: Introduction of a local database (MongoDB/PostgreSQL) for caching SharePoint data, reducing API calls and improving response times.
  • Unit & Integration Testing: Comprehensive test suite using Jest and React Testing Library to ensure reliability.
  • CI/CD Pipeline: Automated build, test, and deployment pipeline using GitHub Actions or Azure DevOps.
  • Production Deployment: Hosting on Azure App Service with proper environment configurations, SSL certificates, and domain mapping.
  • Error Monitoring: Integration of application monitoring tools (e.g., Sentry, Application Insights) for production error tracking.

3. Documentation & Quality Assurance

  • API documentation using Swagger/OpenAPI specifications.
  • User manual and administrator guide preparation.
  • Security audit and vulnerability assessment.
  • Performance benchmarking and load testing.
